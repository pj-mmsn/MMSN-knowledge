# AI 项目面试加分指南

> **一句话**:你有 AI 项目经验，但面试官会追问细节——这些精准回答让你从「了解 AI」变成「落地过 AI」。

## 你的 AI 项目怎么讲（按 STAR 包装）

### 项目一：AI 小说家（桌面 AI 写作工具）

**S**：想做一款桌面端 AI 写作工具，市面上都是网页版，离线体验差。
**T**：从零搭建 PyQt5 桌面应用 + DeepSeek API 集成 + 5 种写作模式。
**A**：
- 用 PyQt5 实现跨平台桌面 GUI，支持暗色主题
- 设计了 Prompt 模板系统，5 种写作模式（小说、诗歌、散文、剧本、对话）共用一套模板引擎
- 踩过 DeepSeek 双协议兼容的坑——OpenAI 和 Anthropic 两种协议端点、格式、鉴权方式全不同
**R**：完整交付，从 Prompt 设计到 GUI 到打包，掌握 AI API 集成全流程。

**面试话术**：「做这个项目的最大收获不是代码，而是 Prompt 工程——同样的模型，Prompt 设计好坏能差出一个数量级。我们用了角色设定 + 输出格式约束 + Few-shot 示例三层 Prompt 结构。」

### 项目二：从零构建 Agent

**S**：市面上 Agent 框架太多，想知道底层到底怎么工作的。
**T**：从零搭建一个支持工具调用的 Agent，不用 LangChain 等框架。
**A**：
- 实现了 ReAct 循环（Thought → Action → Observation）
- 自建 Function Calling 适配层，统一了 OpenAI 和 Anthropic 的调用格式
- 加了一套参数校验 + 重试机制，防止 LLM 输出非法 JSON 导致崩溃
**R**：深入理解了 Agent 的底层机制，能讲清楚 LangChain/LangGraph 在做什么

**面试话术**：「Agent 听起来神秘，本质就是一个循环——LLM 输出要调什么工具，代码去执行，结果再喂给 LLM。我不用框架手写了一遍之后，再去看 LangChain 源码就能看懂了。」

### 项目三：RAG 知识库（最近在做的）

**S**：团队内部文档散落在多个地方，新人 onboarding 成本高。
**T**：搭建一个内部知识库问答系统。
**A**：
- Chunking 策略迭代了 4 版——从固定 500 字到按 Markdown 标题层级切分 + 20% 重叠
- 检索从纯向量相似度升级到混合检索（向量 + BM25）+ Rerank 重排序，准确率提升 15%
- 接入 Spring AI，Java 生态集成零学习成本
**R**：新人问接口文档之类的问题减少了 60%。

**面试话术**：「RAG 的核心不在模型，在检索。Chunking 切多大、要不要重叠、用什么 Embedding 模型、要不要 Rerank——每个选择都比模型本身影响更大。」

## 被追问的精准回答

### Q: RAG 和微调什么时候用哪个？

> 「90% 场景用 RAG——成本低、秒级更新、数据不出服务器。微调只在 RAG 不够时用——比如需要模型学会特定输出格式或行业术语。」

### Q: Agent 开发你踩过什么坑？

> 「LLM 输出的 JSON 有时候格式不对，直接 parse 会崩。解决方案是两层防御——外层 try-catch 兜底，内层用 JSON Schema 约束 + retry。还有一个坑是一个 task 里连续调 10+ 次工具会上下文溢出，加了一个中间摘要机制。」

### Q: 怎么评价一个 LLM 的好坏？

> 「分场景。通用对话看回复质量和幻觉率，代码生成看是否可编译运行，Function Calling 看 JSON 格式正确率和参数准确率。不会只看榜单，一定用自己业务的数据评测。」

### Q: Java 和 Python AI 生态怎么选？

> 「Python 做 AI 实验和原型最快，生态最全。但 Java 系统集成 AI 我倾向 Spring AI——团队都是 Java 开发，零门槛上手。我的策略是 Python 做探索，Java 做落地。」
