# 位运算基础与实战

> **一句话**：位运算比四则运算快一个数量级。HashMap 的槽位定位、线程池的状态打包、JDK 的权限位掩码——全是位运算在干活。

> 📌 关联阅读：[HashMap](HashMap.md) · [线程池](线程池.md) · [ConcurrentHashMap](ConcurrentHashMap.md)

## 六大位运算符

| 运算符 | 名称 | 运算规则 | 口诀 |
|:--:|------|---------|------|
| `&` | 与 AND | 全 1 得 1 | 两个真才是真 |
| `\|` | 或 OR | 有 1 得 1 | 一个真就够 |
| `~` | 非 NOT | 1 变 0，0 变 1 | 取反 |
| `^` | 异或 XOR | 不同得 1 | 一样是 0，不一样是 1 |
| `<<` | 左移 | 高位丢弃，低位补 0 | ×2 的 n 次方 |
| `>>` | 算术右移 | 高位补符号位 | ÷2 的 n 次方（有符号） |
| `>>>` | 逻辑右移 | 高位补 0 | 无符号右移 |

```java
int a = 5;   // 0101
int b = 3;   // 0011

a & b   // 0001 = 1    (两个位都是1才得1)
a | b   // 0111 = 7    (有1就得1)
a ^ b   // 0110 = 6    (不同得1)
~a      // 1010 = -6   (按位取反)
a << 1  // 1010 = 10   (左移1位 = ×2)
a >> 1  // 0010 = 2    (右移1位 = ÷2)
```

## HashMap 中的位运算

```java
// ① 桶下标定位：用 & 代替 %
//    n 是 2 的幂时，(n-1) 二进制全是 1
//    1101 0110 (214)
//  & 0000 1111 (15)
//    ──────────
//    0000 0110 (6)    ← 相当于 hash % 16

// ② 扰动函数：高位参与低位计算
(h = key.hashCode()) ^ (h >>> 16)
// 高16位和低16位做XOR → 增大随机性 → 减少冲突

// ③ 扩容判断：hash & oldCap
//    oldCap = 16 = 0b10000
//    hash 的第5位是0 → 留原位
//    hash 的第5位是1 → 下标 + oldCap
```

## 线程池中的位运算（CTL 位打包）

```java
// 一个 int 的前3位存状态，后29位存线程数
private final AtomicInteger ctl = new AtomicInteger(ctlOf(RUNNING, 0));

private static final int COUNT_BITS = Integer.SIZE - 3;  // 29
private static final int CAPACITY   = (1 << COUNT_BITS) - 1; // 0b000111...1

// 取状态（高3位）
private static int runStateOf(int c) { return c & ~CAPACITY; }

// 取线程数（低29位）
private static int workerCountOf(int c) { return c & CAPACITY; }

// 打包：状态 | 数量
private static int ctlOf(int rs, int wc) { return rs | wc; }
```

## 权限位掩码

```java
// Linux 文件权限：rwx = 111 = 7
//   owner=7, group=5, other=4  →  0754
int READ   = 1 << 2;  // 100 = 4
int WRITE  = 1 << 1;  // 010 = 2
int EXEC   = 1 << 0;  // 001 = 1

int permission = READ | WRITE;  // 110 = 6 (可读可写)
boolean canRead  = (permission & READ)  != 0;  // true
boolean canExec  = (permission & EXEC)  != 0;  // false
```

## 面试常考题

```java
// 判断奇偶：比 n%2 快
boolean isOdd  = (n & 1) == 1;
boolean isEven = (n & 1) == 0;

// 交换两个数（不用临时变量）
a ^= b; b ^= a; a ^= b;  // XOR 特性：a^b^b=a

// 判断是否是 2 的幂
boolean isPowerOf2 = (n > 0) && (n & (n - 1)) == 0;
// 8 = 1000, 7 = 0111 → 1000 & 0111 = 0000 ✅
// 6 = 0110, 5 = 0101 → 0110 & 0101 = 0100 ❌

// 乘除 2 的 n 次方
int   mul  = n << 3;      // n × 8
int   div  = n >> 3;      // n ÷ 8
int   mod  = n & 7;       // n % 8  (仅当除数是 2 的幂)

// 统计 1 的个数（JDK Integer.bitCount 源码）
int count = Integer.bitCount(n);
```

## 面试话术

「位运算在 JDK 源码里几乎无处不在。HashMap 的 `(n-1)&hash` 是一条 AND 指令代替除法，线程池的 ctl 用 int 的高 3 位存状态低 29 位存数量——这些设计让 Java 的核心类在高并发场景下几乎零开销。理解位运算才能真正看懂 JDK 源码。」
