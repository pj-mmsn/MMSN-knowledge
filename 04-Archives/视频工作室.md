## 2026-07-04 多模型视频工作室 — 按特长分配模型的设计实践

**场景**：构建一个多模型协作的视频制作流水线——推理模型写剧本、图像模型画分镜、视频模型制成片。无需单个全能模型，而是让每个模型做最擅长的事。

**架构：顺序管道模式（知识库 Multi-Agent 四大模式之一）**

```
用户创意
   ↓
[Stage 1] Director (推理模型)
   模型: GPT-4o / DeepSeek-R1
   输入: 一句话想法 + 视觉风格
   输出: 结构化剧本（5-8个场景，每个含视觉/动效提示词）
   ↓
[Stage 2] Storyboard (图像生成模型)
   模型: DALL-E 3 / Stable Diffusion
   输入: 剧本中的 visual_prompt
   输出: 5-8 张分镜图（每张对应一个场景）
   ↓
[Stage 3] Videographer (视频生成模型)
   模型: RunwayML Gen-3 / Pika / Sora
   输入: 分镜图 + motion_prompt
   输出: 视频片段
   ↓
最终成片
```

**关键设计决策**：

1. **结构化中间产物**：每个阶段输出结构化的 `dataclass`（Script→Scene, Storyboard→Shot, VideoClip），阶段之间通过数据结构解耦，任意阶段可独立替换模型。

2. **Mock 模式设计**：每个 Agent 都支持 `use_mock` 标志。在没有 API Key 时：
   - Director：返回模板化示例剧本（5段起承转合结构）
   - Storyboard：Pillow 生成带提示词标注的占位图
   - Videographer：复制分镜图作为视频帧 + JSON 元数据
   这样无需任何 API Key 即可跑通全流程。

3. **模型替换原则**：每个 Agent 通过构造函数注入 `Client`，而非硬编码模型名。切换模型只需改配置：
   ```python
   pipeline = VideoPipeline(
       director_model="deepseek-r1",  # 从 gpt-4o 换到 deepseek
       image_model="stable-diffusion-xl",
   )
   ```

4. **Director 提示词工程**：系统提示词定义了严格 JSON 输出格式 + 创作要求（起承转合、记忆点、视觉提示词用英文）。这是保证下游能消费产物的关键。

**测试验证**：
- Mock 模式测试了 2 个创意："小猫东京寻路(anime)" + "宇航员火星发现花(cinematic)"
- 输出 10 张分镜占位图 + 视频元数据 + 完整剧本 JSON

**结论**：
1. 多模型协作的核心不是模型本身，而是**标准化中间格式**——只要上下游约定好数据结构，模型可以任意替换
2. Mock 模式是 AI 项目的必备测试手段——在没有 API Key 时验证架构正确性
3. 导演的 System Prompt 质量直接决定整个流水线的产出一致性
4. 真实 API 时建议用不同的 API Key 分别配置三个 Agent（避免单个模型的 rate limit 拖慢全流程）

## 2026-07-04 v2.0 全维度优化 — 知识库驱动的迭代实践

**场景**: 对 v1.0 做深度代码审查，对照知识库找到 9 个优化点，全部实施后的经验总结。

**优化清单与关键学习**：

### P0: 持久化 + 上下文
- **SQLite 每项目独立 DB**：6 张表（projects/scenes/shots/clips/reviews/story_bible），每个 Stage 完成自动写入
- **断点续传**：`get_progress()` 检测 → 跳过已完成 Stage → 从断点继续
- **StoryBible**：LLM 从每场景提取角色/地点/道具 → 注入后续 Prompt → 场景 50 和场景 1 保持一致

### P0: 审查 + 视频合成
- **Reviewer（CrewAI 反馈循环）**：每 Stage 后评分，<6 分自动重试（最多 3 次）
- **视频合成**：moviepy 图片序列 → MP4 + 硬字幕。依赖缺失自动降级为图片序列
- **旁白**：edge-tts 将 dialogue 转语音，依赖缺失跳过

### P1: 工程化
- 全项目 `print()` → `logging`（控制台 + 文件双输出）
- CLI 支持命令行参数（`--idea --style --mock --resume`）
- Mock 剧本从用户 idea 提取关键词动态填充（不再返回固定模板）

### P2: 代码质量
- Lambda 闭包空值问题（`lambda: review(production.script)` → `lambda result: review(result)`）
- DB 连接生命周期管理（`close()` 由调用方控制，不在 `produce()` 内关闭）

**最终架构**：Director → Review → Storyboard → Review → Videographer → Review → Composer（4 Stage + 3 审查）
