# Spring 启动报错

启动类相关报错的归档。

---

## 2026-07-02 Failed to configure a DataSource

**场景**:Spring Boot 项目启动,报 `Failed to configure a DataSource`。

**根因**:项目引入了数据库相关的 starter(如 `spring-boot-starter-data-jpa` 或 MyBatis),Spring Boot 自动配置触发 `DataSourceAutoConfiguration`,但 pom 里漏了数据库驱动依赖,导致找不到驱动类,自动配置失败。

**结论**:在 `pom.xml` 里补上对应数据库的驱动依赖。MySQL 用:

```xml
<dependency>
    <groupId>com.mysql</groupId>
    <artifactId>mysql-connector-j</artifactId>
    <scope>runtime</scope>
</dependency>
```

**排查思路**(下次遇到同类报错可复用):看到这个报错先查三处 ——
1. pom 里有没有数据库 starter(jpa / mybatis / jdbc)
2. pom 里有没有对应数据库驱动
3. `application.yml` 的 `spring.datasource.url / username / password` 配没配、对不对

如果项目暂时不需要数据库,也可以在启动类排除自动配置作为临时绕过:
```java
@SpringBootApplication(exclude = {DataSourceAutoConfiguration.class})
```
